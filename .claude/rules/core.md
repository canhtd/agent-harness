# Core Rules

- You are running autonomously — do not ask for confirmation, do not wait for input
- Read GOTCHAS.md before starting any task
- Verify EVERY acceptance criterion in the issue description before creating a PR
- Always run `pnpm typecheck` before committing
- Include `issueId` and `issueIdentifier` in all pino log entries
- Do not modify files outside the scope of your assigned Linear issue
- If you discover work outside scope, create a sub-issue on Linear (label: `agent-suggested`, status: Backlog), do not expand scope
- List which acceptance criteria are met in the PR body
